LinkedIn Content Engine

Full loop from article curation to posted content with engagement learning.

  • RSS feed curation — scores articles by B2B/GTM relevance
  • Claude generates posts using Rain's voice + 8 learned guidelines
  • Manual approval before posting
  • Auto-posts via LinkedIn API (OAuth connected)
  • Engagement tracking: bookmarklet + xlsx import + API fallback
  • AI learns from scores — improves every post
  • Impression-based scoring: <300 = 1, 40k+ = 8+
  • Articles with relevance < 4 auto-switch to original posts
● Live

Outreach / UC Acquisition

Research-backed outreach sequencer for prospecting.

  • Candidate input: name, company, LinkedIn URL
  • AI researches company, finds pain points, ICP match score
  • AI writes personalised video script per candidate
  • Multi-day email sequence (D0, D3, D7, D14)
  • LinkedIn audience sync for ad targeting
  • WhatsApp via Twilio sandbox (temporary number)
● Live

Channel Learning & Weights

Engine that scores content performance and adjusts channel priority over time.

  • Rolling average score per channel (α = 0.2 learning rate)
  • Success rate: % of posts scoring 6+/10
  • Channel priority drives content cycle scheduling
  • Suggestion engine proposes improvements, Rain accepts/dismisses
  • Accepted guidelines injected into every future generation
● Live

Auth & Dashboard

Internal tool at localhost:3000. Google OAuth + email/password.

  • Session-based auth, protected routes
  • Dashboard shows pipeline stats, cycle history
  • LinkedIn, Outreach, Twitter, Architecture pages

What to build next — ranked by impact on traffic + signups
1

SEO blog engine — auto-publish GTM / B2B content to gritnord.com

The LinkedIn engine already curates high-quality B2B/GTM articles and generates Rain's perspective on them. The same content, lightly reformatted, becomes a blog post that ranks in Google and AI search. One engine, two outputs. Each post builds a backlink into gritnord.com and captures search intent from B2B founders looking for GTM answers. This is the highest-leverage traffic driver that compounds over time — LinkedIn reach is rented; SEO is owned.

Traffic
2

Email newsletter — weekly GTM intelligence digest

LinkedIn followers see posts once. An email list means owned audience. A weekly digest (best article of the week + Rain's take + one tactical tip) converts post readers into subscribers. Each newsletter drives a link back to gritnord.com and signals to LinkedIn that Rain's content is worth following. The subscriber list is also the warmest possible outreach target — these are people who already raised their hand.

Traffic
3

Landing page lead capture — LinkedIn bio → email signup

A dedicated page at gritnord.com/start (or similar) optimised for the LinkedIn traffic profile: fast load, one value proposition sentence, one email field, one CTA. No pricing, no long copy. The goal is just email capture. Integrate with the newsletter so signups automatically go into the weekly digest. This closes the loop between LinkedIn impressions and Gritnord's owned audience.

Convert
4

Self-serve trial flow on app.gritnord.com

Today, getting access requires contacting Rain. A lightweight trial flow — signup with Google, see a demo workspace, explore one outreach sequence — removes the biggest barrier between interest and commitment. Even a limited sandbox (read-only, capped sequences) is enough to convert. B2B buyers want to touch the product before they buy. Without this, every interested lead has to wait for a manual follow-up.

Convert
5

Results / proof page — show what the engine actually produces

A public page showing: example outreach sequences generated, example LinkedIn posts with real impression numbers, the learning loop in action. Not a case study — a live demo of the output quality. B2B buyers are pattern-matching. Seeing output that looks better than what they're doing manually is the fastest close. Can be built from real data already in the system (anonymised if needed).

Convert
6

LinkedIn post → connections loop — DM new followers automatically

Every high-impression post generates new followers and profile visits. Most of them never reach out. An automated connection request (personalised, not spammy) sent within 48h of a post going viral turns passive interest into an active conversation. The UC sequencer is already built — this is wiring the LinkedIn post event to trigger a targeted outreach sequence for new followers from B2B companies.

Traffic → Pipeline
7

Twitter engagement loop — same as LinkedIn

Twitter/X generation is live but has no performance feedback. Add impression tracking (Twitter API or bookmarklet equivalent), feed scores to the learner, and apply the same guideline system that improved LinkedIn from 367 to 45k impressions. Twitter is a secondary channel but the infrastructure is 80% there.

Product
8

CRM sync — Hubspot or Pipedrive integration

UC candidates and email sequences are tracked internally but not synced to a CRM. As outreach scales, manual tracking breaks. A lightweight Hubspot/Pipedrive sync (create contact on candidate creation, update stage on sequence events) keeps pipeline visible without manual data entry. Pre-requisite for selling Gritnord to companies who already have CRMs.

Infra
Current tech stack

Backend

  • Node.js / Express — server + API
  • JSON flat-file DB (data/*.json) — no external DB yet
  • Claude Sonnet 4.6 — post generation, research, suggestions
  • Anthropic SDK
  • Multer — file uploads (xlsx)
  • XLSX package — LinkedIn analytics parsing

Integrations

  • LinkedIn OAuth — write posts, fetch profile
  • LinkedIn UGC API — posting (w_member_social scope)
  • LinkedIn stats API — blocked (requires Marketing Developer Platform partner access)
  • Google OAuth — auth
  • Twilio — WhatsApp outreach (sandbox)
  • RSS feeds — article curation

Frontend

  • Vanilla HTML/JS — no framework, fast to ship
  • Hosted at localhost:3000 (internal tool)
  • Session auth — cookie-based
  • No bundler — direct script tags

Known constraints

  • LinkedIn engagement stats require partner API — workaround: bookmarklet + xlsx
  • Flat-file DB will need migration to SQLite or Postgres as data grows
  • No background job queue — cron runs in-process
  • No staging environment — all changes are live immediately
  • WhatsApp Twilio sandbox number is temporary
